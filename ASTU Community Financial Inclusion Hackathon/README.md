# Financial Inclusion in Africa — Hackathon

## What is this project about?

This project tries to **predict whether a person in Africa has a bank account or not** using information about their life, like their job, age, education, and where they live.

Having a bank account is a big deal — it helps people save money, make payments, and access loans. But in many African countries, most people still don't have one. This model helps understand **who is most likely to be financially included** and why.

Countries covered: **Kenya, Rwanda, Tanzania, and Uganda**

---

## The Goal

Build a machine learning model that predicts:

- `1` → The person **has** a bank account
- `0` → The person **does not** have a bank account

The competition is scored using **Mean Absolute Error (MAE)** — the lower your score, the better.

---

## The Data

There are 6 files in this project:

| File | What it contains |
|---|---|
| `Train.csv` | 23,524 rows of labeled data used to train the model |
| `Test.csv` | 10,086 rows where you must predict the answer |
| `SampleSubmission.csv` | An example of what your submission file should look like |
| `submission.csv` | The submission file generated by this project |
| `VariableDefinitions.csv` | Explains what each column means |
| `Financial_Inclusion.ipynb` | The main notebook with all the code |

---

## The Features (Columns)

These are the pieces of information available for each person:

| Column | What it means |
|---|---|
| `country` | Which country the person is from |
| `year` | The year the survey was done (2016, 2017, or 2018) |
| `uniqueid` | A unique ID for each person |
| `location_type` | Rural or Urban |
| `cellphone_access` | Does the person have a phone? (Yes / No) |
| `household_size` | How many people live in their home |
| `age_of_respondent` | The person's age |
| `gender_of_respondent` | Male or Female |
| `relationship_with_head` | Their role in the household (e.g. Head, Spouse, Child) |
| `marital_status` | Married, Single, Widowed, etc. |
| `education_level` | Highest level of school completed |
| `job_type` | Type of work they do |

> **Target column (what we predict):** `bank_account` — Yes or No

---

## Quick Data Facts

- **Training data:** ~23,524 people
- **Test data:** ~10,086 people
- About **86%** of people in the training data do **not** have a bank account — so the dataset is imbalanced. Keep this in mind when building your model.

1. Make sure you have Python installed with these common libraries:
   - `pandas`

2. Open the notebook:
   ```
   Financial_Inclusion.ipynb
   ```

3. Run all the cells from top to bottom.

4. A `submission.csv` file will be created at the end.

---

## Submission Format

Your submission file must look like this:

```
unique_id,bank_account
uniqueid_1 x Kenya,1
uniqueid_2 x Kenya,0
uniqueid_3 x Tanzania,1
```

- Use the exact `unique_id` values from `Test.csv`
- Predict `1` (has bank account) or `0` (does not)

---

## Evaluation Metric

**Mean Absolute Error (MAE)**

This measures how far off your predictions are on average. A score of `0` would be perfect. The lower your MAE, the higher you rank on the leaderboard.
